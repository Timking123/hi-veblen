## 测试执行

### 前提条件
1. 安装 Playwright: `npm install -D @playwright/test`
2. 安装浏览器: `npx playwright install`
3. 确保开发服务器可在端口 5173 启动

### 运行测试

```bash
# 运行所有测试
npm run test:e2e

# 在 UI 模式下运行测试
npm run test:e2e:ui

# 在有头模式下运行测试（可见浏览器）
npm run test:e2e:headed

# 运行特定浏览器的测试
npm run test:e2e:chromium
npm run test:e2e:firefox
npm run test:e2e:webkit

# 运行特定测试文件
npx playwright test e2e/navigation.spec.ts

# 调试测试
npx playwright test --debug
```

### 替代方案: 系统 Chrome
如果 Playwright 浏览器安装失败，使用系统 Chrome：

```bash
npx playwright test --config=playwright.config.system-chrome.ts
```

## 测试设计原则

### 1. 优雅降级
测试设计为优雅处理可选元素：
- 交互前检查元素是否存在
- 对可选功能使用条件逻辑
- 可选元素缺失时不失败

### 2. 稳定选择器
测试使用多种选择器策略：
- data-testid 属性（首选）
- 语义选择器（文本内容）
- CSS 类（备选）
- 基于角色的选择器

### 3. 适当等待
测试使用适当的等待策略：
- `waitForLoadState('networkidle')` 用于页面加载
- `waitForTimeout()` 用于动画
- 大多数操作使用 Playwright 内置自动等待

### 4. 跨浏览器支持
测试配置为在以下浏览器运行：
- 桌面: Chromium, Firefox, WebKit
- 移动端: Chrome (Pixel 5), Safari (iPhone 12)

### 5. 响应式测试
测试在多个断点验证布局：
- 桌面: 1920×1080
- 平板: 768×1024
- 移动端: 375×667

## 已知限制

### 浏览器安装
- WebKit 可能在 Windows 上不可用
- 移动端 Safari 需要 WebKit
- 网络问题可能阻止浏览器下载
- 解决方案: 使用系统 Chrome 和替代配置

### 测试环境
- 测试需要开发服务器运行
- 端口 5173 必须可用
- 首次运行某些测试可能较慢（冷启动）

### 可选功能
- 某些测试检查可选元素（表单、图表）
- 即使可选元素缺失测试也会通过
- 这允许实现的灵活性

## 持续集成

### CI 配置
测试为 CI 环境配置：
- 失败时自动重试（2 次重试）
- 单工作进程以保证稳定性
- HTML 报告生成
- 失败时捕获截图和追踪

### GitHub Actions 示例
```yaml
- name: 安装依赖
  run: npm ci

- name: 安装 Playwright 浏览器
  run: npx playwright install --with-deps

- name: 运行 E2E 测试
  run: npm run test:e2e

- name: 上传测试结果
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: playwright-report
    path: playwright-report/
```

## 维护

### 添加新测试
1. 在 `e2e/` 目录创建新的 `.spec.ts` 文件
2. 遵循现有测试结构
3. 添加需求验证注释
4. 使用描述性测试名称
5. 优雅处理可选元素

### 更新测试
1. 保持选择器稳定（优先使用 data-testid）
2. UI 变更时更新测试
3. 保持需求可追溯性
4. 在所有配置的浏览器上测试

### 调试失败
1. 在有头模式运行: `npm run test:e2e:headed`
2. 使用调试模式: `npx playwright test --debug`
3. 检查 `test-results/` 中的截图
4. 查看 `playwright-report/` 中的追踪
