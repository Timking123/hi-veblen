## 资源评估与建议

### 适用场景

该服务器配置（2核2G）适合：
- ✅ 轻量级 Web 应用托管
- ✅ 个人作品集网站
- ✅ 小型后台管理系统
- ✅ API 服务（低并发）
- ⚠️ 中等流量网站（需监控）

### 资源限制与优化建议

#### CPU (2 vCPU)
- 足够运行 Node.js/Python 后端服务
- 建议使用 PM2 进行进程管理，限制 Node.js 实例数
- 避免 CPU 密集型操作（如大量图片处理）

#### 内存 (2 GB)
- **关键限制**：需要精细管理内存使用
- 建议分配：
  - 系统预留：~500 MB
  - Nginx：~50 MB
  - Node.js 后端：~300-500 MB
  - MySQL/SQLite：~200-300 MB
  - 前端静态文件缓存：~200 MB
- 优化措施：
  - 使用 SQLite 替代 MySQL（节省内存）
  - 配置 Node.js 内存限制：`--max-old-space-size=512`
  - 启用 Swap 分区（建议 2-4 GB）

#### 存储 (40 GB ESSD)
- 足够存储网站文件、数据库、日志
- 建议分配：
  - 系统：~10 GB
  - 应用代码：~5 GB
  - 数据库：~5 GB
  - 日志：~5 GB
  - 上传文件：~10 GB
  - 预留空间：~5 GB
- 定期清理日志和临时文件

#### 带宽 (3 Mbps)
- 理论下载速度：~375 KB/s
- 适合文本内容为主的网站
- 优化措施：
  - 启用 Gzip/Brotli 压缩
  - 使用 CDN 加速静态资源
  - 图片压缩和懒加载
  - 考虑使用阿里云 OSS 存储大文件

## 推荐技术栈

### 后端
- **运行时**：Node.js 20 LTS（内存占用较低）
- **框架**：Express.js 或 Fastify（轻量高效）
- **数据库**：SQLite（无需额外进程，节省内存）
- **进程管理**：PM2

### 前端
- **构建工具**：Vite（生成优化的静态文件）
- **部署方式**：Nginx 直接托管静态文件

### Web 服务器
- **Nginx**：反向代理 + 静态文件服务 + Gzip 压缩

### 可选优化
- **CDN**：阿里云 CDN 加速静态资源
- **OSS**：大文件存储（简历、图片等）
- **SSL**：Let's Encrypt 免费证书

## 安全建议

1. **防火墙配置**
   - 仅开放必要端口：22(SSH)、80(HTTP)、443(HTTPS)
   - 配置安全组规则限制 SSH 访问 IP

2. **SSH 安全**
   - 禁用密码登录，使用密钥认证
   - 修改默认 SSH 端口
   - 安装 fail2ban 防止暴力破解

3. **应用安全**
   - 后台管理系统添加登录认证
   - API 接口添加速率限制
   - 定期更新系统和依赖包

4. **数据备份**
   - 定期备份数据库和上传文件
   - 考虑使用阿里云快照功能

---

## 监控建议

1. **系统监控**
   - 使用阿里云云监控
   - 设置 CPU/内存/磁盘告警阈值

2. **应用监控**
   - PM2 监控 Node.js 进程
   - 配置日志轮转（logrotate）

3. **关键指标**
   - CPU 使用率 > 80% 告警
   - 内存使用率 > 85% 告警
   - 磁盘使用率 > 80% 告警
   - 带宽使用率监控

## 注意事项

1. **内存是主要瓶颈**：2GB 内存需要精细管理，避免内存泄漏
2. **带宽有限**：3Mbps 适合轻量应用，大文件建议使用 CDN/OSS
3. **长期稳定**：服务器有效期至 2028 年，适合长期项目
4. **弹性扩展**：如流量增长，可升级配置或增加带宽
